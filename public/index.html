Git gotchas

Branching workflow as per adeleandluke.com, as I can never remember.

Option eines – Branch sauce

  1. Make branch
  2. Do some work
  3. Stage work
  4. Commit work
  5. Push branch
    • Type: 
      git push -u origin branch-name
    • Gives URL to open pull req
  6. Make pull request
  7. Check preview environment and wait for test to pass
  8. Merge if OK
  9. Check production
  10. Make woohoo
  11. Back to term
  12. Checkout to master branch
  13. Type: 
    git pull
  14. And then run this to tidy up old branches: 
    git springclean

Option zwei – Branch like you mean it

  1. Do some work
  2. Stage it
  3. Commit it
  4. Push while making a branch:
    • Type: 
      git push origin HEAD:branch-name
    • Gives URL to open pull req
  5. Make pull request
  6. Check preview environment and wait for test to pass
  7. Merge if OK
  8. Check production
  9. Make woohoo
  10. Back to term
  11. Run: 
    git pull
  12. Run: 
    git springclean

Advantages to using your own environment

As follows:

  • The amazing shortcuts possible on an editor! Makes light work, saves such a lot of time and you'll wonder how you ever did it so slowly before.
  • You may as well get started in the working process you will eventually be using. Why not get aqcuainted right away and hit the ground running? Customising it will make it fit like a glove too.
  • Because you can give full screen treatment to every resource you're working with: your editor, .css file, browser window, etc. It makes sense to have your viewport at a max of 100% with each and not be using a damn letterbox of an editor like on Codepen. Let it breathe, baby!

Setting up an app with node.js and Express

Prerequisite

If you don't have Homebrew installed yet, open your terminal and paste this: 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"
The script will explain what it'll do and will pause before it does it. Once that's done its thing, move on to the below.

Steps

Type in your terminal: 

brew install node
Once that's all done then type: 
mkdir app_name
cd app_name
Now to initialise the project and link it to npm: 
npm init
The terminal will prompt you for a few things, you can just enter your way through it – however, when you get to this: 
entry point: (index.js)
Change it to this: 
app.js
Next, run this to install Express (make sure you're still within the app_name directory): 
 npm install express --save
Open up your editor and create a file named app.js within the app_name directory. Then write the following: 
var express = require('express');
var app = express();

app.get('/', function (req, res) {
  res.send('Hello! I am birthed!');
});

app.listen(3000, function () {
  console.log('Example app listening on port 3000!');
});
Now, you can get the app running. Go back to the terminal and run the command: 
node app.js
Once you've done that, you can access via http://localhost:3000/ in your browser address bar. Boom.
