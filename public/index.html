Git gotchas

Branching workflow as per adeleandluke.com, as I can never remember.

Option eines – Branch sauce

  1. Make branch
  2. Do some work
  3. Stage work
  4. Commit work
  5. Push branch
    • Type: 
      git push -u origin branch-name
    • Gives URL to open pull req
  6. Make pull request
  7. Check preview environment and wait for test to pass
  8. Merge if OK
  9. Check production
  10. Make woohoo
  11. Back to term
  12. Checkout to master branch
  13. Type: 
    git pull
  14. And then run this to tidy up old branches: 
    git springclean

Option zwei – Branch like you mean it

  1. Do some work
  2. Stage it
  3. Commit it
  4. Push while making a branch:
    • Type: 
      git push origin HEAD:branch-name
    • Gives URL to open pull req
  5. Make pull request
  6. Check preview environment and wait for test to pass
  7. Merge if OK
  8. Check production
  9. Make woohoo
  10. Back to term
  11. Run: 
    git pull
  12. Run: 
    git springclean

Advantages to using your own environment

As follows:

  • The amazing shortcuts possible on an editor! Makes light work, saves such a lot of time and you'll wonder how you ever did it so slowly before.
  • You may as well get started in the working process you will eventually be using. Why not get aqcuainted right away and hit the ground running? Customising it will make it fit like a glove too.
  • Because you can give full screen treatment to every resource you're working with: your editor, .css file, browser window, etc. It makes sense to have your viewport at a max of 100% with each and not be using a damn letterbox of an editor like on Codepen. Let it breathe, baby!
